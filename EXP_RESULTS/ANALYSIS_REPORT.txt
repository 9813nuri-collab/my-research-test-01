[논문 프레이밍]

  "The Faithfulness Paradox of Exact Implementation"

  수식을 정확히 실행하는 것(exact implementation)과
  전문가의 의도를 충실히 따르는 것(intent faithfulness)은
  같은 것이 아니다.

  수식 엔진은 다단계 보정(win_rate, decay, regime, override)을
  정확히 실행하지만, 이 보정들이 원래 전문가의 "이 지표가 좋으면
  HIGH" 라는 단순한 의도를 왜곡한다.

  LLM은 텍스트로 표현된 "의도"를 직관적으로 따르므로
  의도 충실도(intent faithfulness)가 더 높다.

[수정된 논문 내러티브]

  Title: "The Faithfulness Paradox: When Exact Implementation
         Diverges from Expert Intent in Multi-Agent Systems"

  Core Thesis:
  "수식을 정확히 실행하는 것(computational faithfulness)과
   전문가의 의도를 충실히 따르는 것(intent faithfulness)은
   근본적으로 다른 차원이다.

   구조화된 온톨로지 엔진은:
   (a) 예측 가능한 인과적 행동(TAF 80%)을 보이지만
   (b) 다단계 파이프라인이 원래 의도를 왜곡하며(OFS 0.458)
   (c) 강한 Override 메커니즘이 다양성을 소멸시킨다(CDI 0.167)

   LLM 페르소나 엔진은:
   (a) 높은 의도 충실도(OFS 0.818)를 보이지만
   (b) 인과적 해석 가능성이 없으며
   (c) temperature=0에서도 예측 불가능한 내부 논리를 가진다

   이 Trade-off는 '어떤 방식이 더 좋은가?'가 아니라
   '어떤 상황에서 어떤 충실도가 중요한가?'라는 질문을 열어준다."

  Part 10. Ground Truth 한계 및 보완 방안
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[현재 Ground Truth의 한계]

  Ground Truth v2.0은 "단순 임계값 해석"에 기반:
    "지표가 임계값을 넘으면 → 이 전문가는 HIGH 점수를 줄 것"

  그러나 실제 엔진은 더 복잡한 파이프라인을 거침:
    Step 3: × win_rate (0.54~0.99) × decay_factor (0.83~1.0)
    Phase 1: × override_factor (~0.004 when Taleb fires)
    Phase 2: Global Adjustment (macro weight)
    Phase 3: Synergy/Suppress

  결과: Ground Truth가 "수식이 HIGH을 줄 것"이라 예측한 경우에도
  실제 파이프라인은 multiplicative corrections로 인해 LOW를 줌.

[보완 방안 (향후 작업)]

  방안 A: 파이프라인 기반 Ground Truth (Pipeline-Derived GT)
    → 실제 엔진 파이프라인 (override 없이)의 출력으로 Ground Truth 재도출
    → C1_fair OFS = 1.0 (by definition)이 되므로 의미 없음

  방안 B: 적응적 임계값 (Adaptive Threshold)
    → C1 점수 분포의 중앙값을 threshold로 사용
    → 분포별 상대적 OFS를 측정

  방안 C: 현재 접근을 유지하되 해석을 전환
    → "Ground Truth = 전문가 의도의 단순화된 표현"으로 정의
    → OFS = "의도 충실도"로 해석 (수식 정확도가 아님)
    → 이것이 가장 정직하고 흥미로운 프레이밍

  ★ 권장: 방안 C → 논문에서 Ground Truth의 한계를 투명하게 기술하고
     "The Faithfulness Paradox"를 핵심 기여로 삼음

[OFS Section Strategy]

  서술 순서:
  1. C1(override 포함) vs C2: C2가 더 높음 → 표면적으로 놀라운 결과
  2. Override 제거 후 C1_fair: 더 낮아짐 → 파이프라인 복잡성이 원인
  3. Ground Truth의 성격 설명: "의도 충실도" 측정임을 명확화
  4. 결론: "수식 정확성 ≠ 의도 충실성" — The Faithfulness Paradox

  핵심 수치:
    C2 OFS = 0.818 > C1 OFS = 0.632 > C1_fair OFS = 0.458
    해석: LLM이 전문가의 "무엇을 해야 하는지"를 더 잘 이해

[CDI Section Strategy]

  서술 순서:
  1. C1(override) CDI = 0.167 → "Override가 다양성을 파괴"
  2. C1_fair CDI = 0.706 ≈ C2 CDI = 0.723 → "공정한 조건에서 동점"
  3. C3_fair CDI = 0.683 < C1_fair CDI = 0.706 → "Edge가 차별화에 기여"
  4. 결론: "구조와 LLM 모두 유사한 수준의 전문가 분화를 달성 가능"

[TAF Section Strategy — 논문의 핵심 무기]

  서술 순서:
  1. 5개 사전 예측 중 4개 정확 (80%)
  2. no_taleb: 22.8배 앙상블 증가 → 극적인 인과적 예측
  3. no_risk: 전체 리스크 도메인 제거 → 예측된 분산 증가
  4. no_edges 실패 분석: Override 환경에서의 한계 설명
  5. 결론: "구조화된 시스템만이 이런 인과적 예측을 가능케 함"

  반론 방어:
    "C2(LLM)에서 한 페르소나를 제거하면 어떻게 되는지 사전 예측 가능한가?"

[핵심 발견 1: 상전이 현상 (Phase Transition)]

  k=0.5 → k=1.0에서 OFS가 0.9236 → 0.6319로 급락.
  이것은 "상전이(phase transition)"와 유사한 현상:

    k < 1.0: Override가 온건하여 전문가 점수의 HIGH/LOW 방향이 보존됨
             → OFS 0.92 (intent faithfulness 유지)
    k ≥ 1.0: Override가 충분히 강해져 점수가 임계값 아래로 압축됨
             → OFS 0.63 (의도 왜곡 시작)

  논문 의의: 구조화된 시스템에서 단일 파라미터(k)가
  "의도 보존 영역"과 "의도 왜곡 영역"의 경계를 결정한다.
  이것은 neuro-symbolic 시스템에서 하이퍼파라미터의
  해석적 중요성을 실증한다.

[핵심 발견 2: OFS 고원 현상 (OFS Plateau)]

  k ≥ 1.0에서 OFS가 0.6319으로 완전히 일정.
  이유: k가 증가하면 점수 크기(magnitude)는 계속 압축되지만,
  HIGH/LOW 분류(threshold 0.5 기준)는 이미 k=1.0에서 결정됨.
  더 큰 k는 이미 threshold 아래로 내려간 점수를 더 낮출 뿐.

[핵심 발견 3: CDI의 단조 감소]

  CDI가 k 증가에 따라 체계적으로 감소:
    k=0.5: CDI=0.631 (건강한 분화)
    k=3.5: CDI=0.167 (극단적 동질화)
    k=4.0: CDI=0.135 (최저점)

  예외: k=5.0에서 CDI=0.208로 소폭 반등.
  원인: 극단적 압축(mean=0.014) 환경에서 전문가간
  미세한 차이(0.001~0.03)가 상대적으로 큰 비율 차이가 됨.

[핵심 발견 4: 최적 k값 도출]

  k=0.5일 때 OFS=0.92, CDI=0.63으로 모든 메트릭 최적.
  현재 기본값 k=3.5는 Override 강도가 과도함.

  그러나 k=0.5가 "최적"이라고 논문에서 주장하기보다는:
  "k값에 따라 시스템의 faithfulness-diversity tradeoff가
  결정적으로 변화한다"는 해석적 발견으로 프레이밍하는 것이 적절.

[C1_fair 점수 분포]

  min=0.3486, max=0.9802

  즉 0.5 threshold는 C1_fair에게 "너무 낮아서" 거의 모든 점수가

[핵심 발견: The Faithfulness Paradox의 역전!]

  고정 임계값 0.5: C2(0.818) >> C1_fair(0.458) — C2 승리
  적응 임계값 0.633: C3_fair(0.879) > C1_fair(0.823) > C2(0.776) — C1/C3 승리!

  패러독스가 **역전**되었다:
  - 임계값을 C1_fair의 점수 분포에 맞추면
    구조화된 엔진이 LLM보다 높은 OFS를 달성함
  - C3_fair(Edge 없음)가 가장 높은 OFS = 0.879
  - 이것은 Edge 상호작용 없이 순수 공식만으로도
    높은 intent faithfulness를 달성할 수 있음을 의미

  3. 따라서 "어느 것이 더 충실한가?"보다

  → "충실도는 맥락 의존적이며, 이 의존성 자체가 발견"

[TAF 직접 비교: C1 (80%) vs C2 (33%)]

  이것은 논문의 핵심 무기가 될 수 있는 결과:
  구조화된 시스템(C1)은 5개 중 4개 예측 성공,
  LLM 시스템(C2)은 3개 중 1개만 성공.

[실패 분석: 왜 C2에서 예측이 실패하는가?]

  1. c2_no_taleb 실패:
     C1에서는 Taleb 제거 → Phase 1 override 해제 → 모든 점수 상승.
     이것은 "시스템 상호작용"에 의한 효과.

     C2에서는 Taleb이 독립적 LLM 호출일 뿐이다.
     Taleb을 제거해도 다른 11명의 점수에는 영향이 없다.
     ensemble이 감소한 이유: 9개 시나리오에서 Taleb 점수 > 다른 전문가 평균.
     즉 높은 점수 기여자를 제거하면 ensemble이 감소.

     핵심: C1에서 Taleb 제거의 효과는 "상호작용에 의한 해방"이고,
     C2에서는 "단순한 기여자 제거". 이 차이가 예측 실패의 근본 원인.

  2. c2_no_risk 실패:
     C1에서는 RSK 전원 제거 → override 해제 → 점수 분산 극적 증가.
     C2에서는 3명 제거 → 12명→9명으로 줄어듦 → 남은 9명의 합산.

     점수 분산이 감소한 이유:
     리스크 전문가들(Taleb, Shannon, Thorp)은 시나리오별로
     극단적 점수(0.0 또는 1.0)를 자주 부여 → 분산에 기여.
     이들을 제거하면 나머지 전문가들의 더 온건한 점수만 남아
     오히려 분산이 감소.

  3. c2_no_graham 성공:
     이것은 가장 "단순한" 예측: Graham은 Value 도메인의
     핵심 기여자이므로 제거 시 A3_TOXIC의 Value 점수가 감소.
     이 예측은 상호작용 효과 없이 직접 기여에만 의존하므로
     C2에서도 성립.

[심층 통찰: 상호작용 효과의 존재 여부가 예측 가능성을 결정]

  C1 TAF 성공 예측들의 공통점:
    → 모두 "시스템 상호작용"(override, synergy, suppress)에 기반

  C2 TAF 실패 예측들의 공통점:
    → C1의 상호작용 효과를 C2에 적용했지만,
      C2에는 그런 상호작용이 존재하지 않음

  C2 TAF 유일한 성공:
    → 상호작용이 아닌 "직접 기여"에 기반한 예측

  결론: 구조화된 온톨로지의 핵심 가치는
  "전문가 간 상호작용(interaction)"을 명시적으로 모델링하여
  그 효과를 사전 예측 가능하게 만드는 것이다.
  LLM 시스템에서는 전문가들이 독립적으로 실행되므로
  상호작용 기반 예측이 본질적으로 불가능하다.
